## STEP 1: Claude Code 설치

### 1-1. 터미널 열기
- **Spotlight 검색**: `Cmd + Space` → "터미널" 입력 → Enter
- 또는 **Finder** → 응용 프로그램 → 유틸리티 → 터미널

검은 창이 뜨면 성공!

### 1-2. Claude Code 설치하기
터미널에 아래를 **그대로 복사 붙여넣기**하고 Enter:

```bash
curl -fsSL https://cli.anthropic.com/install.sh | sh
```

설치가 끝나면 터미널을 **닫았다가 다시 열기**.

### 1-3. 설치 확인
터미널에 입력:

```bash
claude --version
```

버전 번호가 나오면 성공! (예: `1.0.xx`)

### 1-4. 로그인
터미널에 입력:

```bash
claude
```

브라우저가 열리면서 로그인 화면이 뜹니다.
평소 claude.ai에서 쓰는 계정으로 로그인하면 됩니다.
로그인 성공하면 터미널에 Claude Code가 활성화됩니다.

> 💡 처음 실행하면 이것저것 물어볼 수 있는데, 기본값(Enter)으로 넘기면 됩니다.

**Claude Code를 잠시 종료하세요** — `Ctrl + C` 또는 `/exit` 입력

## STEP 4: Claude Code 실행 + 작업 시키기

### 4-1. 프로젝트 폴더에서 Claude Code 실행
터미널에:

```bash
cd ~/Desktop/naesinfit-tests
claude
```

### 4-2. 아래 메시지를 그대로 복사해서 Claude Code에 붙여넣기

```
내신핏_3차_컨텍스트.md 파일과 naesinfit-final.html 파일을 읽어주세요.

⚠️ 중요: naesinfit-final.html은 UI/UX/기능이 모두 확정된 프로토타입입니다.
디자인, 레이아웃, 색상, 애니메이션, 카피를 절대 변경하지 마세요.

해야 할 것:
1. naesinfit-final.html을 index.html로 복사하세요
2. index.html에서 내장된 QUESTION_BANK과 FULL_PASSAGE를 제거하고,
   기존 HTML 파일에서 fetch → 파싱하는 방식으로 교체해주세요.
   (파싱 로직은 컨텍스트 문서에 검증된 코드가 있습니다)
3. QUESTION_BANK이 없을 때는 폴백으로 내장 데이터를 사용하도록 해주세요.
   (GitHub Pages에서는 fetch, 로컬 미리보기에서는 내장 데이터)
4. manifest.json을 생성해주세요
5. 교과서/영어1/비상홍/1과/ 안의 테스트 HTML 파일 3개에서
   const EI와 const Q가 제대로 파싱되는지 확인해주세요.
   워크북테스트.html에는 FULL_PASSAGE 전역 변수가 있으니 함께 추출해주세요.
```

### 4-3. Claude Code가 작업하는 것을 지켜보기
- 파일을 읽고, 수정하고, 만들어줍니다
- 중간에 "이렇게 할까요?" 물어보면 **y** 입력하고 Enter
- 완료되면 `index.html`과 `manifest.json`이 생성됩니다

## ❓ 문제가 생기면

| 상황 | 해결 |
|------|------|
| Claude Code 설치 안 됨 | 터미널 닫았다 다시 열고 재시도 |
| `claude` 명령어 안 됨 | 터미널 새로 열기, 또는 `source ~/.zshrc` |
| 로컬 서버에서 안 보임 | 주소가 `http://localhost:8080` 맞는지 확인 |
| fetch 에러 | 파일 경로에 공백이 있는지 확인 (공백 금지) |
| GitHub push 실패 | Claude Code에게 "push 에러가 났어. 해결해줘" 라고 하기 |
| 디자인이 바뀜 | Claude Code에게 "프로토타입 디자인을 유지해줘" 라고 하기 |

3. 학생 프로필 (이름/학교/학년, localStorage 저장, 로그아웃)
4. 홈 대시보드 (학습현황 통계, 최근기록 5건, 테스트 유형 카드)
5. 교과서 범위 선택 (CSS Grid 아코디언, 칩 선택, DOM 직접 조작)
6. 시험 설정 (테스트 유형 3종 라디오, 문항 수 슬라이더)
7. 시험 화면 (20분 카운트다운, dot 네비, 이전/다음/제출 항상 표시)
8. 채점 (MC + 서술형, trim + 중복공백 정규화 + 대소문자 무시)
9. 결과 화면 (점수 카운트업 애니메이션, 80점↑ confetti)
10. 기본정보 / 레벨카드 / 유형별분석 / 학습플랜 / 액션버튼 / 해설
11. 결과 복사 (텍스트), 이미지 저장 (Canvas PNG 카드 레이아웃)
12. 시험 기록 저장 (localStorage, 최근 20회) + 재시험
13. 해설: 단어테스트(korean/analysis/tip) + 워크북/퀴즈(ref/correct/wrongs/tip) 모두 호환

## Claude Code에서 해야 할 것

### 1순위: fetch 파싱 로직 연결
현재 프로토타입은 데이터가 QUESTION_BANK에 내장되어 있음.
실제 배포 시에는 기존 HTML 파일에서 데이터를 fetch → 파싱해야 함.

**파싱 로직 (검증 완료):**
```javascript
function parseTestHTML(htmlText) {
  // EI 추출
  const eiMatch = htmlText.match(/const\s+EI\s*=\s*(\{[\s\S]*?\});/);
  let ei = null;
  if (eiMatch) {
    try { ei = (new Function('return ' + eiMatch[1]))(); } catch(e) {}
  }
  
  // Q 배열 추출 — 매칭 브라켓 찾기
  const qIdx = htmlText.search(/const\s+Q\s*=\s*\[/);
  let questions = [];
  if (qIdx >= 0) {
    const bracketStart = htmlText.indexOf('[', qIdx);
    let depth = 0, end = bracketStart;
    for (let i = bracketStart; i < htmlText.length; i++) {
      if (htmlText[i] === '[') depth++;
      else if (htmlText[i] === ']') depth--;
      if (depth === 0) { end = i + 1; break; }
    }
    const qStr = htmlText.slice(bracketStart, end);
    try { questions = (new Function('return ' + qStr))(); } catch(e) {}
  }
  return { ei, questions };
}
```

**주의: 워크북테스트에 `FULL_PASSAGE` 전역 변수가 있음**
→ Q 데이터에서 `passage: FULL_PASSAGE`로 참조함
→ 파싱 시 FULL_PASSAGE도 함께 추출하여 eval 컨텍스트에 포함시켜야 함

**파일 경로 빌더:**
```
교과서: 교과서/{교과명}/{출판사폴더명}/{N과}/{테스트유형}.html
모의고사: 모의고사/{학년}/{시험명}/{N번}/{테스트유형}.html
```

### 공통 구조 (3종류 모두 동일)
```javascript
const EI = { subject, pub, lesson, title, total, time, histKey, qCount };
const Q = [
  {
    id, type, diff, pts, fmt, // 공통
```
